# 计算机架构能力认证清单

> 基于《Computer Architecture: A Quantitative Approach》第 6 版
> 完成 36 周课程后的自我认证

## 使用说明

逐项检查，标记你确信已掌握的技能。
- [ ] = 尚未掌握
- [x] = 已掌握，可以应用和解释

---

## Phase 1: 性能基础

- [ ] 能够应用 Amdahl 定律计算加速比和优化上限
- [ ] 能够使用 CPU 性能公式（Iron Law）分解性能瓶颈
- [ ] 能够解释 MIPS/MFLOPS 的局限性，并正确使用 SPEC 基准测试
- [ ] 能够计算和解释成本-性能-功耗的权衡关系

## Phase 2: 指令级并行

- [ ] 能够描述经典 5 级流水线各阶段及其数据通路
- [ ] 能够识别 RAW/WAR/WAW 数据冒险并设计旁路/转发方案
- [ ] 能够分析控制冒险并评估分支预测策略（BTB, Two-Level, Tournament）
- [ ] 能够追踪指令通过 Scoreboarding 算法的执行过程
- [ ] 能够追踪指令通过 Tomasulo 算法的执行过程（含 ROB）

## Phase 3: 超标量与推测执行

- [ ] 能够解释多发射处理的分类（Static vs Dynamic, Superscalar vs VLIW）
- [ ] 能够分析推测执行的收益与 Spectre/Meltdown 安全代价
- [ ] 能够设计并分析分支目标缓冲器（BTB）和返回地址栈（RAS）
- [ ] 能够比较不同值预测技术的精度和覆盖率

## Phase 4: 存储层次

- [ ] 能够设计缓存层次结构并定量分析性能（AMAT 计算）
- [ ] 能够区分 3C 模型中的 Compulsory/Capacity/Conflict Miss
- [ ] 能够比较不同缓存替换策略（LRU, RRIP, SRRIP, DIP）
- [ ] 能够解释虚拟内存的工作原理（TLB, Page Table Walk, 大页）
- [ ] 能够分析存储系统的可靠性（RAID 级别、ECC）

## Phase 5: 并行架构

- [ ] 能够编写和优化 CUDA Kernel（共享内存、Warp Shuffle、Bank Conflict）
- [ ] 能够分析 GPU 的 SIMT 执行模型和 Warp 调度
- [ ] 能够实现和验证缓存一致性协议（MSI, MESI, MOESI）
- [ ] 能够比较监听（Snooping）和目录（Directory）协议的可扩展性
- [ ] 能够分析片上网络（NoC）拓扑对延迟和吞吐量的影响

## Phase 6: 高级并行

- [ ] 能够解释内存一致性模型（SC, TSO, RMO）及其编程影响
- [ ] 能够分析 SMT（同时多线程）的资源利用率和公平性
- [ ] 能够设计向量处理器并计算向量化加速比
- [ ] 能够比较同步机制（锁, Barrier, TM）的性能开销

## Phase 7: 现代架构

- [ ] 能够计算仓库级计算机的 TCO 和 PUE
- [ ] 能够分析能量比例性和尾延迟对 WSC 的影响
- [ ] 能够解释 TPU 脉动阵列的工作原理和性能模型
- [ ] 能够比较 DNN 加速器的数据流策略（WS, OS, RS）
- [ ] 能够分析 DNN 加速器的数据复用效率和算术强度
- [ ] 能够实现 IEEE 754 浮点加法器并解释舍入模式
- [ ] 能够描述嵌入式处理器的低功耗设计技术
- [ ] 能够概述计算机架构 60 年的关键演进与教训

## Phase 8: 顶点项目

- [ ] 能够独立搭建架构仿真环境（gem5/ChampSim/CUDA/BookSim）
- [ ] 能够设计严谨的定量实验（参数扫描、统计重复、误差线）
- [ ] 能够绘制和分析 Roofline 模型
- [ ] 能够撰写符合学术规范的定量分析报告
- [ ] 能够在 5 分钟内清晰传达技术发现

## 全局能力

- [ ] 能够阅读和理解 ISCA/MICRO/HPCA 等顶会论文的定量分析部分
- [ ] 能够参与芯片架构设计讨论并提出数据驱动的建议
- [ ] 能够将架构概念与其他 CS 领域（OS, Compiler, PL）关联
- [ ] 能够追踪计算机架构领域的最新发展（如 Chiplet, CXL, 3D Stacking）
- [ ] 具备从"直觉驱动"到"数据驱动"的架构设计思维方式
