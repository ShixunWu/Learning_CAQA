# 第23周 自测题：向量处理器深入与综合对比

> 闭卷完成，时间 30 分钟。综合比较与定量分析为主。

## 题目

### Q1：Scatter/Gather 效率分析
稀疏矩阵 \(A\) 的非零元素比例为 \(10\%\)。使用 scatter/gather 指令时，每次 gather 装载 N 个元素中仅 10% 有效。向量处理器 MVL=64。处理 N=10000 个索引的稀疏向量加法需多少轮 gather 操作？若 gather 启动开销 12 周期，每元素 1 周期，求总 gather 时间。

### Q2：向量长度 vs 性能
向量处理器：MVL=64，startup=10，每元素=1。处理 n=30（极短向量）。计算标量循环（每迭代 5 周期）vs 向量（需完整走一遍 strip-mining 流程）哪个更快？

### Q3：编译器向量化依赖分析
```c
for (i=0; i<N; i++)   a[i] = b[i] + c;    // 循环1
for (i=1; i<N; i++)   a[i] = a[i-1] * b[i]; // 循环2
for (i=0; i<N; i++)   a[i] = a[i+N] + b[i]; // 循环3
```
判断每个循环是否可被编译器自动向量化，并说明原因。

### Q4：能效对比
三种方案运行同一矩阵乘（N=512）：
- RVV: 15W × 2.5s = 37.5J, 35.8 GFLOPS
- AVX2: 65W × 0.8s = 52J, 112 GFLOPS
- CUDA: 300W × 0.05s = 15J, 1792 GFLOPS

计算各方案的 GFLOPS/W，并从能耗绝对值最小的角度和能效最高的角度分别判断哪个方案最优。

### Q5：向量掩码效率
向量掩码操作：`VADD.VM V1, V2, V3` 仅对掩码为 1 的元素做加法。若掩码 70% 为 1，MVL=64，n=200，startup=8，每元素=1。计算有效执行时间。若没有掩码（需先 gather 有效元素 → 操作 → scatter），额外开销多少？

### Q6：设计权衡
假设你要设计一款面向 HPC 的新型处理器。有三个方案：
A. 纯向量处理器 (MVL=256, 8 lanes)
B. AVX-512 SIMD + 乱序超标量
C. 集成 GPU SM (类似 AMD APU)

从时钟频率、芯片面积、功耗、编程难度四个维度分析各自的优势。

### Q7：RVV 可变长度优势
传统向量机 MVL=64 固定。RVV 的 vsetvli 允许运行时设定 VL。若应用处理长度为 20/50/100/200 四种不同长度的向量，RVV 比固定长度节省多少无效操作比例（以处理 20 长度为例）？

### Q8：最终综合题
某数据中心需处理大量小矩阵乘法（N=64~256 不等），功耗预算有限（<200W）。从向量/SIMD/GPU 中选择并论证最佳方案，需给出至少两个定量理由。

---

## 答案

### A1
轮次 = ceil(10000/64) = 157 轮。
总时间 = 157 × 12 + 10000 × 1 = 1884 + 10000 = **11884 周期**。
标量版本：10000 × 5 = 50000 周期。加速比 ≈ 4.2×。注意 gather 即使仅 10% 有效仍加载全部 64 个元素，效率损失在于内存带宽。

### A2
标量：30 × 5 = **150 周期**。
向量：1 轮 (ceil(30/64)=1)：1 × 10 + 30 × 1 = **40 周期**。
向量快 3.75×。但注意，短向量时启动开销占比高（10/40=25%），但向量仍然胜出。

### A3
- 循环 1：**可向量化**。`b[i]` 无依赖，`c` 为标量。
- 循环 2：**不可向量化**。`a[i]` 依赖 `a[i-1]`（循环携带的 RAW 真依赖）。
- 循环 3：**不可向量化**（或需谨慎）。`a[i+N]` 可能与 `a[i]` 重叠（别名分析），若 N 很大且编译器无法证明无别名，则保守禁止向量化。

### A4
- RVV: 35.8/15 = **2.39 GFLOPS/W**
- AVX2: 112/65 = **1.72 GFLOPS/W**
- CUDA: 1792/300 = **5.97 GFLOPS/W**

能效最高：**CUDA**。能耗绝对值最小：**CUDA (15J)**。GPU 在大规模并行负载下能耗优势明显。

> **注意**：以上功耗数值（15W/65W/300W）假设为**系统级总功耗**，已包含计算核心、缓存、内存控制器及 DRAM 访存功耗。若功耗数据仅包含计算核心而忽略 DRAM 访存（矩阵乘 N=512 时 DRAM 功耗可占总功耗 20-40%），则 RVV 和 AVX2 的能效优势会被高估，CUDA 因计算密度高、DRAM 占比相对小，受影响较小。能耗对比必须使用包含内存子系统的完整系统功耗数据。

### A5
有效操作数：200 × 0.7 = 140。但硬件仍执行所有元素（仅结果写回被掩码控制）。
时间 = ceil(200/64)×8 + 200×1 = 4×8 + 200 = **232 周期**。
无掩码方案：需 gather 有效元素索引 → pack → 操作 → unpack，额外 scalpel 开销约 50-100 周期。

### A6
| 维度 | A. 向量 | B. AVX-512 | C. 集成 GPU |
|------|--------|-----------|------------|
| 频率 | 低 (~1.5 GHz) | 高 (~4 GHz) | 中 (~2 GHz) |
| 面积 | 中 | 中 | 大 |
| 功耗 | 低 (15-30W) | 中 (65-150W) | 高 (100-300W) |
| 编程难度 | 低 | 中 | 高 |

向量在能效和编程易用性上占优；AVX-512 兼容性好；GPU 峰值高但功耗和编程成本大。

### A7
固定 MVL=64 处理 n=20：1 轮完整 64 元素（44 无效）→ 浪费 = 44/64 = **68.75%**。
RVV 设 VL=20：仅处理 20 元素，浪费 = **0%**。
对于 n=100：固定需 2 轮（64+36，第二轮浪费 28 元素），RVV 可设 VL=100（若 VLMAX≥100）。

### A8
推荐 **向量处理器**（RVV 风格）：
1. 小矩阵（N=64-256）天然适合向量长度范围，GPU 的 kernel launch 开销对此规模较大；
2. 功耗预算 200W 排除了高端 GPU（需 300W+），而向量处理器可在 30-50W 内实现高效处理；
3. 编译器自动向量化成熟，降低开发成本。
