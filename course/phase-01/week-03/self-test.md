# Week 3 自测

## Q1：ISA 分类

以下指令序列分别属于哪种 ISA 类型？
```
A: ADD R1, R2, R3     (寄存器-寄存器)
B: ADD R1, [R2]       (寄存器-内存)
C: PUSH A; PUSH B; ADD; POP C  (栈)
```

## Q2：RISC-V 编码效率

某程序用 RV32I 编译后为 1000 条指令（每条4字节）。用 RV32IC（含压缩指令）编译后，60% 的指令被压缩为 2 字节。代码大小减少了多少？

## Q3：寻址模式

RISC-V 只有 `reg + offset` 一种寻址模式。x86 有 `[base + index*scale + offset]`。x86 的这种复杂寻址模式对哪个 Iron Law 因子有益？对哪个因子有害？

## Q4：设计权衡

如果让你设计一个新的 ISA，在以下方面你会如何决策？为什么？
1. 32个 vs 16个通用寄存器
2. 固定长度 vs 可变长度编码
3. 条件码标志 vs 比较并分支

---

## 答案

### A1
A = Register-Register (Load/Store), B = Register-Memory, C = Stack

### A2
压缩后大小：400×4 + 600×2 = 2800 字节。原大小：4000 字节。减少 30%。

### A3
x86 复杂寻址有益于 **IC**（一条指令完成地址计算+访存），但有害于 **CPI**（译码和执行更复杂）和 **Cycle Time**（关键路径更长）。

### A4
1. **32个更好**：减少 spill/fill（寄存器不够用来回倒腾内存），编译器更容易分配
2. **固定长度更好**：简化译码、便于宽发射。但可变长度代码密度更高
3. **比较并分支更好**：避免条件码成为数据依赖，便于乱序执行
