# Week 4 自测：Phase 1 综合练习

## Q1：综合性能分析

处理器 X 执行程序 P 耗时 10 秒。分析发现：30% 时间在整数运算（CPI=1），40% 在访存（CPI=3），30% 在浮点（CPI=2）。如果改进 Cache 使访存 CPI 降到 1.5，新的执行时间是多少？

## Q2：Amdahl vs Gustafson

某图像处理程序：90% 的像素处理可并行化，10% 是串行步骤。如果在 8 核 CPU 上运行：
- 按 Amdahl 定律，加速比是多少？
- 如果图像分辨率翻 4 倍（问题规模变大），按 Gustafson 定律，加速比是多少？

## Q3：设计决策

你是一个嵌入式 SoC 架构师。应用场景是语音唤醒（always-on, 99% 时间休眠）。你有两个 IP 核可选：
- Core A：1 GHz，0.5mW（休眠），50mW（活跃），CPI=1.5
- Core B：200 MHz，0.01mW（休眠），5mW（活跃），CPI=1.0

哪个更适合？用数字说明。

---

## 答案

### A1
原始平均 CPI = 0.3×1 + 0.4×3 + 0.3×2 = 0.3 + 1.2 + 0.6 = 2.1。
优化后 = 0.3×1 + 0.4×1.5 + 0.3×2 = 0.3 + 0.6 + 0.6 = 1.5。
新执行时间 = 10 × 1.5/2.1 ≈ **7.14 秒**。

### A2
Amdahl: Speedup = 1/(0.1 + 0.9/8) = 1/(0.1+0.1125) = 1/0.2125 ≈ **4.7x**。
Gustafson（像素翻4倍→并行工作翻4倍）: Speedup = 0.1 + 0.9×8 = **7.3x**。

### A3
日均功耗：Core A = 0.5×99% + 50×1% = 0.495 + 0.5 = 0.995 mW。
Core B = 0.01×99% + 5×1% = 0.0099 + 0.05 = 0.06 mW。
Core B 功耗仅 6%，且 CPI 更低。选 **Core B**。尽管频率低，但语音唤醒不需要高性能。"Always-on"场景下休眠功耗是决定性因素。
