# Week 16 自测：多发射与 SMT

## Q1：超标量加速比

某超标量处理器发射宽度为 4。理想情况下 IPC=4。实际 benchmark 测得 IPC=1.6。利用率是多少？列举 3 个可能导致 IPC 远低于发射宽度的原因。

## Q2：结构冒险

4 发射处理器有 2 个整数 ALU、1 个 FPU、1 个 Load/Store 单元。以下指令序列每周期能发射几条？
```
add x1,x2,x3    # 整数
add x4,x5,x6    # 整数
add x7,x8,x9    # 整数（第 3 条整数，但只有 2 个 ALU）
fadd f0,f1,f2   # 浮点
lw   x10,0(x11) # 访存
```

## Q3：SMT 原理

SMT 处理器有 2 个线程。线程 A 的 IPC=1.2（单线程时），线程 B 的 IPC=0.8。SMT 综合 IPC 为 1.5。为什么不是 1.2+0.8=2.0？解释 FU 资源竞争。

## Q4：SMT vs 细粒度多线程

以下场景哪种更适合 SMT，哪种更适合细粒度多线程（fine-grained MT）？
A) 两个线程都是计算密集型，几乎不 stall
B) 线程 A 频繁 cache miss（50% 周期 stall），线程 B 计算密集

## Q5：综合计算

4 发射处理器，每个线程的 ILP 为 1.5。SMT 配置 2 线程后 IPC 提升到 2.2。计算：
- 单线程时发射槽利用率
- SMT 时发射槽利用率
- 如果有足够 ILP 的理想线程，最多能利用多少发射槽？

---

## 答案

### A1
利用率 = 1.6/4 = 40%。原因：
1. 数据依赖（RAW）限制了 ILP
2. 分支误预测导致流水线气泡
3. Cache miss 导致长时间 stall
4. ROB/LSQ 大小不够，无法暴露足够的 ILP
5. 结构资源不足（FU 数量少于发射宽度）

### A2
第 1-2 条：2 条整数（OK），浮点 OK（1 条），load OK（1 条）→ 共 4 条，可以发射。
第 3 条整数和第 1-2 条一起：只有 2 个 ALU → 第 3 条整数被阻塞 → 实际发射 3 条（2 整数 + 1 浮点 + 1 load 共 4，但——等等，如果 load 占 LS 单元，fadd 占 FPU，2 个 add 占 2 ALU，正好 4 条全发射）。但第 3 条 add 会结构阻塞，要么前 2 条 add 发射，第 3 条等下一周期。
所以：最多 4 条可以同时发射（2 add + 1 fadd + 1 lw），第 3 个 add 延迟到下周期。

### A3
资源竞争：2 个线程共享 FU、ROB、发射队列等。A 的 ADD 指令可能和 B 的 ADD 竞争 ALU 端口。此外，总发射宽度为 4，两个线程之和不能超 4。FU 争用导致每个线程的实际 IPC 低于单独运行时的 IPC。

### A4
A) 细粒度多线程：SMT 无帮助（两个都不 stall），细粒度 MT 可增加整体吞吐。
B) SMT：线程 A 的大量 stall 周期可被 B 的指令填充，SMT 比细粒度 MT 更灵活（不强制按周期切换）。

### A5
- 单线程：利用率 = 1.5/4 = 37.5%
- SMT：利用率 = 2.2/4 = 55%
- 理想情况：如果有足够多个高 ILP 线程，理论上可接近 100%（实际受 FU 数量限制）
