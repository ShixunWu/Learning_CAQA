## 核心概念

### ILP 的理论极限

Wall (1991) 的研究结论：
- 完美分支预测 + 完美 alias 分析下，ILP 可达 7-60（取决于 benchmark）
- 64 个整数 + 64 个浮点寄存器时，实际 ILP 约 7
- 增加更多资源收益递减

### 三大墙

| 墙 | 描述 |
|----|------|
| **功耗墙** | 增加 FU、发射宽度 → 功耗指数增长 |
| **复杂度墙** | 发射宽度每翻倍，wakeup/select 逻辑复杂度平方增长 |
| **ILP 墙** | 程序中可用的 ILP 本身就有限（依赖链、分支） |

### Phase 4 知识框架

```
静态调度 → 动态调度 → 推测执行 → 多发射 → VLIW
  5级      记分板    Tomasulo   超标量    EPIC
  │          │          │+ROB      │        │
  │          │          │          │        │
  └── 顺序 ──┴── 乱序执行 ──┴── 推测 ──┴── 多发射 ──┘
```

| 技术 | 解决什么 | 代价 |
|------|----------|------|
| Forwarding | RAW（部分）| 硬件前推路径 |
| Scoreboard | 乱序执行 | 三张表 + WAR阻塞 |
| Tomasulo | WAW/WAR | 保留站 + CDB |
| ROB | 推测+精确异常 | ROB 存储 |
| Superscalar | 每周期多条 | 复杂发射逻辑 |
| SMT | 填充空闲槽 | 多线程资源 |
| VLIW | 简化硬件 | 编译器复杂度 |
