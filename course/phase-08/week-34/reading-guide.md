# Week 34 阅读指南：顶点项目选题与设计

> 课程阶段：Phase 8 Capstone Project | 核心主题：项目选择与可行性评估

## 学习目标

- 理解五个可选的顶点项目方向及其技术挑战
- 掌握每个项目所需的工具链和先修知识
- 学会撰写项目提案并进行可行性评估

## 五个顶点项目选项

### 项目 1：超标量 RISC-V CPU（gem5 建模）

**目标**：在 gem5 模拟器中实现一个超标量 RISC-V 处理器模型，探索不同的微架构参数
（发射宽度、ROB 大小、分支预测器）对性能的影响。

| 项目要素 | 说明 |
|---------|------|
| 工具 | gem5 模拟器、RISC-V GCC 工具链 |
| 先修知识 | Phase 2-3（流水线、Tomasulo、超标量） |
| 核心任务 | 配置 O3CPU 模型、运行 SPEC CPU 基准测试、参数扫描 |
| 预期产出 | 性能-面积-功耗权衡分析报告 |
| 难度 | ★★★★☆ |

### 项目 2：多级缓存优化

**目标**：使用 ChampSim 或 gem5 的缓存模型，针对 SPEC CPU 2017 基准测试设计并优化
多级缓存层次结构。

| 项目要素 | 说明 |
|---------|------|
| 工具 | ChampSim / gem5 Ruby |
| 先修知识 | Phase 4（缓存、预取、3C 模型） |
| 核心任务 | 缓存大小/关联度/替换策略对比、预取器设计 |
| 预期产出 | 缓存配置优化方案和 Miss Rate 分析 |
| 难度 | ★★★☆☆ |

### 项目 3：GPU Kernel 优化竞赛

**目标**：针对矩阵乘法、卷积、规约等算子，在 CUDA 上实现并优化，比较不同优化策略
（共享内存、warp shuffle、bank conflict 避免）的效果。

| 项目要素 | 说明 |
|---------|------|
| 工具 | NVIDIA CUDA Toolkit、Nsight Compute |
| 先修知识 | Phase 5（GPU、SIMT、CUDA） |
| 核心任务 | 实现 3+ 个 kernel variants，性能分析和优化 |
| 预期产出 | 每个 kernel 的 roofline 分析、优化日志 |
| 难度 | ★★★★☆ |

### 项目 4：片上网络（NoC）拓扑对比

**目标**：使用 BookSim 或 gem5 Garnet 评估 Mesh、Torus、Flattened Butterfly 等拓扑
在不同流量模式下的延迟和吞吐量表现。

| 项目要素 | 说明 |
|---------|------|
| 工具 | BookSim 2.0 / gem5 Garnet |
| 先修知识 | Phase 5（NoC、互连网络） |
| 核心任务 | 配置拓扑参数、注入率扫描、热点分析 |
| 预期产出 | 延迟-吞吐量曲线、拓扑对比表格 |
| 难度 | ★★★☆☆ |

### 项目 5：DNN 加速器数据流设计

**目标**：使用 MAESTRO 或自建 Python 模型，探索不同数据流（WS/OS/RS/N 输出）在
ResNet-50/Transformer 等模型上的性能和能耗表现。

| 项目要素 | 说明 |
|---------|------|
| 工具 | MAESTRO / Timeloop / 自定义 Python |
| 先修知识 | Phase 7（DSA、数据流、MAC 阵列） |
| 核心任务 | 数据流映射、能耗/延迟建模、设计空间探索 |
| 预期产出 | 数据流对比分析、Pareto 最优设计点 |
| 难度 | ★★★☆☆ |

## 如何选择项目？

| 兴趣方向 | 推荐项目 |
|---------|---------|
| CPU 微架构 | 项目 1（gem5 RISC-V） |
| 存储系统 | 项目 2（缓存优化） |
| 并行编程 | 项目 3（GPU Kernel） |
| 互连网络 | 项目 4（NoC） |
| AI 硬件 | 项目 5（DNN 加速器） |
