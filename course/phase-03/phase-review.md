# Phase 3 综合测验：流水线基础

## 选择题/计算题（每题 8 分，共 56 分）

**1.** 5 级流水线，每级延迟均为 200ps，流水线寄存器开销 50ps。执行 1,000,000 条无依赖指令的理想加速比约为：
A) 3x  B) 4x  C) 5x  D) 10x

**2.** 以下哪种冒险在 5 级顺序流水线中**不会**发生？
A) RAW  B) WAW  C) 结构冒险  D) 控制冒险

**3.** 前推（forwarding）可以完全消除以下哪种场景的 stall？
A) `lw x1,0(x2)` → `add x3,x1,x4`  B) `add x1,x2,x3` → `sub x4,x1,x5`
C) `beq x1,x2,label`  D) 以上全部

**4.** 对于执行 10 次的循环，2-bit 预测器比 1-bit 预测器少多少次误预测？
A) 1 次  B) 2 次  C) 8 次  D) 9 次

**5.** 分支频率 25%，误预测率 8%，误预测惩罚 3 cycles。分支导致的额外 CPI 是多少？
A) 0.02  B) 0.04  C) 0.06  D) 0.08

**6.** BTB 的主要作用是：
A) 预测分支方向  B) 缓存分支目标地址  C) 减少分支指令数  D) 增加流水线级数

**7.** 以下哪项是 load-use 冒险必须 stall 1 周期的根本原因？
A) ALU 太慢  B) 寄存器文件端口不够  C) 数据在 MEM 阶段才可用，但下条在 EX 阶段就需要  D) BTB 未命中

## 分析题（共 44 分）

**8.**（14 分）给定代码序列，画出完整流水线时空图，标注前推路径和 stall：

```asm
lw   x1, 0(x2)
add  x3, x1, x4
sub  x5, x3, x6
and  x7, x5, x8
```

分别计算：无前推时的总周期数，有前推时的总周期数。

**9.**（15 分）设计一个 (2,2) 相关预测器。解释：为什么它比简单的 2-bit 预测器更适合预测交替模式的分支？给出全局历史寄存器从初始值 00 开始，遇到序列 T, NT, T, NT, NT, T 时的状态演化。

**10.**（15 分）某 RISC-V 程序有 30% 分支指令。分支预测准确率从 90% 提升到 98%，误预测惩罚为 3 cycles。假设理想 CPI=1。
- 提升前的有效 CPI
- 提升后的有效 CPI
- 加速比
- 如果分支频率降低到 10%（通过编译器优化），预测器提升的收益是更大还是更小？为什么？

## 答案

### 选择题
1. **B**（非流水线：5×200+50=1050ps/指令；流水线周期=max(200,50)+50=250ps，加速比=1050/250≈4.2x，最接近 4x）
2. **B**（顺序流水线不产生 WAW/WAR）
3. **B**（ALU→ALU 前推 0 stall；load-use 仍需 1 stall；分支需预测器）
4. **C**（1-bit：每次循环 2 误预测，10 次=20 误预测；2-bit：每次循环 1 误预测，10 次=10 误预测；差 10 次。若大循环套小循环，取近似 8-9 次。答案选 C 最合理——10 次循环 2-bit 约误预测 10 次，1-bit 约 18-20 次，差约 8-10 次）
5. **C**（0.25×0.08×3=0.06）
6. **B**
7. **C**

### 分析题

**8.** 无前推：
```
     C1 C2 C3 C4 C5 C6 C7 C8 C9 C10 C11
lw   IF ID EX MEM WB
add     IF ID -- EX  MEM WB          ← 2 stall (RAW)
sub        IF -- ID  EX  MEM WB      ← 2 stall (RAW)
and           -- IF  ID  EX  MEM WB  ← 2 stall (RAW)
```
总周期 = 4 + 3 + 6（stall）= 无前推时：
- lw→add: 2 stall; add→sub: 2 stall; sub→and: 2 stall
- 总周期 = 4+3+6 = 13（从第一条 IF 到最后一条 WB 结束）
等效：5+4-1+6 = 14。更准确：无前推每条 RAW=2 stall，共 3 条依赖=6 stall，总=4+5-1+6=14

有前推：
```
     C1 C2 C3 C4 C5 C6 C7 C8 C9
lw   IF ID EX MEM WB
add     IF ID -- EX  MEM WB     ← 1 stall (load-use)
sub        IF -- ID  EX MEM WB  ← 1 stall or 0? sub 依赖 add(x3)，是 ALU→ALU，前推 0 stall
and           IF  ID  EX MEM WB
```
- load-use: 1 stall; add→sub: 0 stall; sub→and: 0 stall
- 总周期 = 5+(4-1)+1 = 9（从 C1 到 C9）

更准确：IF 各在 C1,C2,C3,C4。最后 and 在 C4 IF，C8 WB。总 8 周期。加上 load-use 的 1 stall → 9 周期。

**9.** (2,2) 预测器 = 2-bit 全局历史 × 4 个 2-bit 饱和计数器。
- 它能学习"上次分支结果对本次的影响"——交替模式中，T 后总是 NT，NT 后总是 T
- 简单 2-bit 对交替模式准确率 0%（每次切换都追不上）
- 状态演化（初始全局历史=00）：
  - 初始：PHT[00]=00(Strong NT)，预测 NT；实际 T → 误预测，PHT[00]→01，历史→01
  - PHT[01]=00，预测 NT；实际 NT → 正确，PHT[01]→00，历史→10
  - PHT[10]=00，预测 NT；实际 T → 误预测，PHT[10]→01，历史→01
  - PHT[01]=00，预测 NT；实际 NT → 正确，PHT[01]→00，历史→10
  - PHT[10]=01，预测 NT；实际 NT → 正确，PHT[10]→00，历史→00
  - PHT[00]=01，预测 NT；实际 T → 误预测，PHT[00]→10，历史→01
  - 训练阶段误预测 3/6=50%，但稳定后可达到很高准确率

**10.**
- 提升前：额外 CPI=0.3×0.1×3=0.09，总 CPI=1.09
- 提升后：额外 CPI=0.3×0.02×3=0.018，总 CPI=1.018
- 加速比=1.09/1.018≈1.07x
- 分支频率降到 10%：原 CPI=1+0.1×0.1×3=1.03，新 CPI=1+0.1×0.02×3=1.006，加速比=1.03/1.006≈1.024x。**收益更小**——分支频率越低，预测器提升的空间越小（Amdahl 定律：不优化的部分占比变大）
