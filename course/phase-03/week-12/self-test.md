# Week 12 自测：分支预测

## Q1：1-bit vs 2-bit 预测器

一个内循环执行 100 次（分支 taken 99 次，最后一次 not-taken）。分别计算 1-bit 和 2-bit 预测器的误预测次数和准确率。

## Q2：分支惩罚计算

某处理器分支频率 20%，误预测惩罚 2 cycles。如果预测准确率从 90% 提升到 95%，整体加速比是多少？（假设理想 CPI=1）

## Q3：2-bit 状态机

画出 2-bit 饱和计数器的状态转移图。如果当前状态是 Weak Taken (10)，接下来遇到 NT/NT/T/T 四个实际结果，最终状态是什么？过程中的预测各是什么？

## Q4：BTB 作用

BTB（Branch Target Buffer）和分支方向预测器有什么区别？为什么现代处理器两者都需要？

## Q5：相关预测器

对于以下 2 个分支的交替模式，1-bit 和 (1,1) 相关预测器哪个更好？为什么？
```
Branch A: T, NT, T, NT, T, NT ...
Branch B: T, NT, T, NT, T, NT ...
```

## Q6：综合计算

某程序：分支 20%，其中无条件跳转（jump）5%，条件分支 15%。无条件跳转在 ID 阶段解析（1 个 stall），条件分支在 EX 阶段解析（2 个 stall）。分支预测准确率 90%，BTB 命中率 95%。计算分支导致的额外 CPI。

---

## 答案

### A1
- 1-bit：循环内第 1 次 NT→T（误），之后 98 次 T→T（正确），最后 NT→NT（正确），但下次循环第 1 次又是 T→NT（误）。100 次迭代 × 10 = 1000 个分支，每个循环 2 次误预测 → 200/1000 = 80% 准确率
- 2-bit：仅每次循环的最后一次（退出）误预测。100 次循环 = 100 次误预测 → 900/1000 = 90% 准确率

### A2
- 原来：额外 CPI = 0.2×0.1×2 = 0.04，总 CPI=1.04
- 提升后：额外 CPI = 0.2×0.05×2 = 0.02，总 CPI=1.02
- 加速比 = 1.04/1.02 ≈ 1.02x（约 2% 提升）

### A3
状态转移：
- Weak T(10) + NT → Weak NT(01)，预测 T（误）
- Weak NT(01) + NT → Strong NT(00)，预测 NT（正确）
- Strong NT(00) + T → Weak NT(01)，预测 NT（误）
- Weak NT(01) + T → Weak T(10)，预测 NT（误）
最终状态：Weak T(10)

### A4
- 方向预测器：预测 T/NT
- BTB：缓存分支目标地址
- 两者都需：知道跳转（方向预测）但不知道跳到哪（BTB 缺失），仍需等待计算目标地址 → 增加惩罚

### A5
(1,1) 相关预测器更好。1-bit 每次切换都误预测（准确率 0%），但 (1,1) 可用 A 的历史预测 B：A 为 T 时 B 为 NT，A 为 NT 时 B 为 T → 几乎 100% 准确。

### A6
- Jump：5%×1=0.05（全部 1 stall，跳转目标明确）
- 条件分支：15% × (0.05×(2+0) + 0.1×2) 不如直接算：
  - BTB 未命中：0.05 × 额外惩罚
  - 误预测：0.1 × 2 = 0.2
  - 总：0.15 × ((1-0.95)×? + 0.1×2) ≈ 0.15 × (0.05×? + 0.2)
- 简化：条件分支额外 CPI ≈ 0.15 × 0.1 × 2 = 0.03，加上 jump 0.05×1=0.05，总额外 CPI ≈ 0.08
