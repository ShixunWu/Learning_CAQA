# 第25周 阅读指南：缓存一致性协议

> 对应教材：CAQA 6th Edition, Chapter 5 §5.4-5.6

## 学习目标

1. 掌握 Snooping 缓存一致性协议的工作原理
2. 理解 MSI/MESI/MOESI 状态转换及其设计权衡
3. 掌握目录协议（Directory Protocol）在大规模系统中的优势

## 核心概念

### 缓存一致性的两个不变条件

1. **单写多读（SWMR）**：同一时刻，一个地址要么被多个核同时读，要么仅被一个核写
2. **数据传播（Data-Value）**：读操作返回该地址最近一次写的值

### Snooping 协议

**工作原理**：
- 每个缓存控制器监听（snoop）总线上的地址
- 读 miss → 广播到总线，拥有最新副本的缓存响应
- 写 miss → 广播使其他副本失效或更新

**两种策略**：
- **Write-Invalidate**（主流）：写前使其他副本失效
- **Write-Update**（非主流）：写时广播更新值

### MSI 协议（3 状态）

| 状态 | 含义 | 可读 | 可写 |
|------|------|:----:|:----:|
| **M**odified | 仅本缓存有，已被修改 | ✓ | ✓ |
| **S**hared | 多缓存共享，干净 | ✓ | ✗ |
| **I**nvalid | 无效 | ✗ | ✗ |

关键转换：
- Read miss → 发射 BusRd；若有 M 者→降级到 S；进入 S
- Write miss → 发射 BusRdX；其他副本失效；进入 M
- Snoop BusRdX 命中→失效（S→I 或 M→I）

### MESI 协议（4 状态）
增加 **E**xclusive：干净且独占（与 M 的区别：未修改）。优势：从 E 写不需发 BusRdX（静默升级到 M）。

### MOESI 协议（5 状态）
增加 **O**wned：脏但共享（与 M 的区别：共享）。优势：M→O 降级时 Owned 缓存可提供数据给其他读请求（减少内存访问）。

### 目录协议（Directory）

**动机**：Snooping 依赖总线广播，无法扩展到 >8-16 核。

**目录结构**：
- 每个内存块对应一个目录项，记录哪些缓存持有该块的副本
- 三种目录格式：全映射（Full-map）、有限指针（Limited pointer）、粗粒度向量

**典型流程**：
1. 写 miss → 向目录发请求 → 目录查表 → 仅向持有者发失效
2. 多播替代广播 → 带宽与核数成正比而非平方

### 协议比较

| 特性 | Snooping | Directory |
|------|----------|-----------|
| 通信 | 广播（O(n)） | 点对点（O(log n)） |
| 可扩展性 | ≤16 核 | 1000+ 核 |
| 延迟 | 低 | 高（需查目录） |
| 实现复杂度 | 低 | 中-高 |

## 关键计算

**Snooping 广播带宽需求**：
\[
\text{BW} = P \times \text{MissRate} \times \text{CacheLineSize}
\]
