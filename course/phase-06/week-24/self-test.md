# 第24周 自测题：多处理器体系结构

> 闭卷完成，时间 30 分钟。UMA/NUMA 延迟分析与可扩展性计算。

## 题目

### Q1：Amdahl 定律
程序 95% 可并行化，在 64 核处理器上的理论最大加速比是多少？

### Q2：Gustafson 定律
问题规模随处理器数线性增长，串行部分固定为 5%。在 64 核上的加速比？

### Q3：SMP 总线瓶颈
SMP 系统有 8 核，共享总线带宽 64 GB/s。每核对内存带宽需求为 10 GB/s。系统是核计算瓶颈还是总线瓶颈？

### Q4：NUMA 平均延迟
4 节点 NUMA 系统，本地延迟 80ns，远程延迟 250ns。优化良好的程序 95% 本地访问，优化差的程序 60% 本地访问。求两种情况的平均延迟。

### Q5：缓存一致性流量增长
双核时每次 shared write miss 产生 1 次 invalidation + 1 个 response。n 核时产生多少次消息？若 8 核系统 cache miss rate = 2%，shared fraction = 20%，每 1000 次访存产生多少一致性消息？

### Q6：SMP 可扩展上限
若共享总线每秒处理 10^9 次一致性事务，每个核平均每 1000 周期产生 1 次 shared miss。频率 3 GHz。最多支持多少核？

### Q7：UMA vs NUMA 编程模型
UMA 编程不需要关心数据位置，NUMA 需要。说明 NUMA 上错误的 "first-touch" 内存分配可能造成什么性能问题。

### Q8：多核加速比异常
某并行程序在 4 核上加速比 3.8×，8 核上仅 4.2×。请从缓存一致性角度分析可能的原因（给出至少两个）。

---

## 答案

### A1
\( S = \frac{1}{(1-0.95) + 0.95/64} = \frac{1}{0.05 + 0.0148} = \frac{1}{0.0648} \approx \mathbf{15.4\times} \)。

### A2
Gustafson: \( S = 64 - 0.05 \times (64-1) = 64 - 3.15 = \mathbf{60.85\times} \)。

### A3
总需求带宽 = 8 × 10 = 80 GB/s > 64 GB/s。**总线瓶颈**。即使各核计算能力充足，总线限制了整体吞吐量。

### A4
优化良好：\( 0.95 \times 80 + 0.05 \times 250 = 76 + 12.5 = \mathbf{88.5\ ns} \)。
优化差：\( 0.6 \times 80 + 0.4 \times 250 = 48 + 100 = \mathbf{148\ ns} \)。
差 1.67×。

### A5
n 核时 shared write miss：1 invalidation 给 n-1 个其他核 → **n-1 条 invalidate + 1 条 response = n 条消息**。
每 1000 次访存：miss = 20 次，其中 shared = 20×0.2 = 4 次。
消息数 = 4 × 8 = **32 条一致性消息**。

### A6
每核每秒 shared miss = 3 GHz / 1000 = 3×10^6 次。
设最大核数 P，总 miss = P × 3×10^6 ≤ 10^9。
P ≤ 10^9 / (3×10^6) ≈ **333 核**。

### A7
First-touch 策略：数据分配到首次访问它的线程所在的 NUMA 节点。如果初始化线程与后续计算线程不在同一节点，所有数据访问都变成远程访问，延迟显著增加。正确的做法是用 `numactl` 或 OpenMP 的 `proc_bind` 确保亲和性。

### A8
可能原因：
1. **一致性流量爆炸**：8 核时共享变量的 invalidate/update 消息量远超 4 核（O(n) vs O(1)），总线/互连饱和。
2. **False Sharing**：多个线程写入同一 cache line 的不同变量，导致 cache line 在核间 ping-pong。
3. **NUMA 远程访问**：8 核系统可能跨 NUMA 节点，远程延迟增加。
