# Week 8 自测

## Q1：TLB 覆盖范围

64 条 TLB 条目，4KB 页。TLB 能覆盖多少 KB？如果工作集是 1MB，预计 TLB miss 率如何？

## Q2：大页代价

某程序分配 2.1 MB 内存。如果使用 2MB 大页，需要几个页？实际分配多少内存（考虑内部碎片）？

## Q3：地址翻译流程

简述 CPU 执行 `ld a0, 0(a1)` 时，虚拟地址到物理地址的完整翻译流程（假设 TLB hit）。

---

## 答案

### A1
覆盖 64 × 4KB = 256KB。工作集 1MB >> 256KB → TLB 覆盖不足，大量 TLB miss。

### A2
需要 ceil(2.1/2) = 2 个页。实际分配 4MB，内部碎片 1.9MB（几乎 50% 浪费）。

### A3
1. CPU 发出虚拟地址（a1 的值）
2. TLB 查询：虚拟页号 → TLB 命中 → 获取物理页号
3. 物理地址 = 物理页号 + 页内偏移
4. 用物理地址访问 L1 Cache
5. 如果 L1 miss → 发到下级存储
