# Week 9 自测：存储与可靠性

## Q1：RAID 容量计算

8 块 4TB 硬盘，分别用 RAID 0/1/5/6，可用容量各是多少？

## Q2：SSD 写放大

某 SSD 的 Flash 块大小为 4MB，页大小为 4KB。应用写入 100 个 4KB 随机写入，每个写入修改不同的逻辑页但它们恰好分布在 5 个Flash块中。假设 SSD 采用"读-修改-写"策略，写放大因子（WAF）是多少？

## Q3：CACTI 结果分析

根据 CACTI 建模结果，32KB→64KB Cache 容量翻倍，访问时间从 1.0ns 增加到 1.3ns，功耗从 0.5nJ 增加到 0.8nJ。如果 miss rate 从 5% 降到 3%，主存访问 50ns/10nJ。从 AMAT 和能耗角度，值得升级吗？

---

## 答案

### A1
- RAID 0: 8×4 = 32TB
- RAID 1: 32/2 = 16TB
- RAID 5: (8-1)×4 = 28TB
- RAID 6: (8-2)×4 = 24TB

### A2
每个 4KB 写入需要擦除 4MB 块（1024 个页）。5 个块 × 4MB = 20MB 物理写入。应用写入 100×4KB = 400KB。WAF = 20MB/400KB = **50x**。

> 这就是为什么 SSD 需要 FTL 和 Wear Leveling —— 随机小写入的写放大非常可怕。

### A3
32KB AMAT = 1.0 + 0.05×50 = 3.5ns。能耗 = 0.5 + 0.05×10 = 1.0nJ。
64KB AMAT = 1.3 + 0.03×50 = 2.8ns。能耗 = 0.8 + 0.03×10 = 1.1nJ。
AMAT 提升 20%，能耗增加 10%。如果性能是首要目标→值得升级。如果是功耗敏感场景→需权衡。
